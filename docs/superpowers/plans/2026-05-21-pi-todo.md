# Pi Todo Extension Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `config/shared/.pi/agent/extensions/todo.ts` as a complete local replacement for `npm:@juicesharp/rpiv-todo`.

**Architecture:** Implement one standalone Pi extension with a pure reducer, in-memory store, branch replay from `todo` tool results, `/todos` command, and above-editor overlay. Keep the public tool schema compatible with `rpiv-todo` while avoiding all `@juicesharp/*` dependencies.

**Tech Stack:** Pi extension API, `@earendil-works/pi-tui`, `typebox`, Node/TypeScript, orgm-dot.

## Task 1: Add todo core and tool

**Files:**
- Create: `config/shared/.pi/agent/extensions/todo.ts`

- [ ] **Step 1: Implement core types, schema, reducer, and `todo` tool**

Create `config/shared/.pi/agent/extensions/todo.ts` with:

- imports from `@earendil-works/pi-coding-agent`, `@earendil-works/pi-ai`, `@earendil-works/pi-tui`, and `typebox` only.
- `TaskStatus`, `TaskAction`, `Task`, `TaskState`, `TaskDetails`, `TaskMutationParams` types.
- TypeBox schema matching `rpiv-todo` input fields.
- in-memory state: `let state: TaskState = { tasks: [], nextId: 1 }`.
- pure helper functions:
  - `cloneState(input)`
  - `visibleTasks(input, includeDeleted?)`
  - `detectCycle(tasks)`
  - `validateBlockedBy(taskId, blockedBy, tasks)`
  - `applyMutation(current, action, params)`
- `pi.registerTool({ name: "todo", ... })`.

Reducer behavior:

- `create`: requires `subject`; creates pending task with next id; validates initial `blockedBy`.
- `update`: requires `id`; refuses deleted task updates; supports `subject`, `description`, `activeForm`, `owner`, `metadata`, `status`, `addBlockedBy`, `removeBlockedBy`; validates transitions and dependency cycles.
- `list`: returns visible tasks, filtered by `status` if supplied, excluding deleted unless `includeDeleted`.
- `get`: requires `id`; returns one task or error.
- `delete`: requires `id`; sets status to `deleted`.
- `clear`: resets to empty state and `nextId: 1`.
- When a task becomes `in_progress`, any other in-progress task moves back to pending.

Tool result shape:

```ts
{
  content: [{ type: "text", text: summary }],
  details: { action, params, tasks: state.tasks, nextId: state.nextId, error? }
}
```

## Task 2: Add replay, command, and overlay

**Files:**
- Modify: `config/shared/.pi/agent/extensions/todo.ts`

- [ ] **Step 1: Add branch replay**

Add `replayFromBranch(ctx)` and call it from `session_start`:

- Scan `ctx.sessionManager.getBranch()`.
- Find latest message entry where `entry.message.role === "toolResult"`, `entry.message.toolName === "todo"`, and `entry.message.details.tasks` / `nextId` exist.
- Replace in-memory state with that snapshot.
- If no snapshot exists, reset to empty state.

- [ ] **Step 2: Add `/todos` command**

Register command:

```ts
pi.registerCommand("todos", {
  description: "Show all todos on the current branch, grouped by status",
  handler: async (_args, ctx) => { ... }
});
```

Behavior:

- If no visible tasks, notify `No todos yet.`
- Otherwise notify grouped sections: Pending, In Progress, Completed.
- Include ids, status icon, subject, owner when present, and blockedBy when present.

- [ ] **Step 3: Add overlay widget**

Add an overlay controller in the same file:

- `installOverlay(ctx)` registers `ctx.ui.setWidget("orgm-todos", factory, { placement: "aboveEditor" })` when visible tasks exist.
- `refreshOverlay()` requests render after tool mutations.
- Hide widget when no visible tasks.
- Render max 12 lines:
  - Heading: `Todos (completed/total)`.
  - Task lines with icons `○`, `◐`, `✓`.
  - `+N more` line when truncated.

Call overlay refresh after:

- `session_start`
- each successful or error tool mutation
- `agent_start` if completed tasks should be hidden after the next response; if not implementing transient hiding, leave completed tasks visible and keep code simple.

Run:

```bash

```

Expected: extension loads without import/runtime errors.
