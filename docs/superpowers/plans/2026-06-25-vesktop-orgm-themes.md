### Anti-patterns
- Do NOT add vesktop to `liveReloadCommands()` — Vencord has its own file-watcher
- Do NOT hardcode hex colors — use `t.Base`, `t.Text`, etc. from Theme struct (same pattern as all other renderers)
- Do NOT touch `cmd/orgm-themes/main.go` — it just calls `BuildWrites()`, no changes needed

### Allowed APIs (from render.go)

## Phase 2: render.go — Add renderVesktop Function

**File:** `internal/orgmtheme/render.go`

### Task 2a: Add the renderVesktop function

Add after the last `func render...` block (before `cssColor` helper). The CSS must override Discord's custom properties for both old `.theme-dark/.theme-light` class-based vars and newer `:root` vars used in recent Discord builds.

```go
func renderVesktop(t Theme) string {
	forceClass := ".theme-dark"
	if t.ColorScheme == "prefer-light" {
		forceClass = ".theme-light"
	}
	return fmt.Sprintf(`/* Generated by orgm-theme. Do not edit. */
/* Force correct theme class and override Discord CSS custom properties. */
%s,
.theme-dark,
.theme-light {
  --background-primary: #%s;
  --background-secondary: #%s;
  --background-secondary-alt: #%s;
  --background-tertiary: #%s;
  --background-floating: #%s;
  --channeltextarea-background: #%s;
  --background-modifier-hover: rgba(%s, 0.16);
  --background-modifier-active: rgba(%s, 0.24);
  --background-modifier-selected: rgba(%s, 0.32);
  --background-modifier-accent: #%s;
  --text-normal: #%s;
  --text-muted: #%s;
  --text-link: #%s;
  --interactive-normal: #%s;
  --interactive-hover: #%s;
  --interactive-active: #%s;
  --interactive-muted: #%s;
  --header-primary: #%s;
  --header-secondary: #%s;
  --channels-default: #%s;
  --channel-text-area-placeholder: #%s;
  --brand-experiment: #%s;
  --brand-360: #%s;
  --status-positive-text: #%s;
  --status-warning-text: #%s;
  --status-danger-text: #%s;
  --scrollbar-thin-thumb: #%s;
  --scrollbar-thin-track: transparent;
  --scrollbar-auto-thumb: #%s;
  --scrollbar-auto-track: #%s;
}
`, forceClass,
		t.Base, t.Mantle, t.Crust, t.Crust, t.Mantle, t.Surface0,
		hexRGB(t.Surface0), hexRGB(t.Surface0), hexRGB(t.Surface0), t.Surface1,
		t.Text, t.Subtext0, t.Blue,
		t.Text, t.Subtext1, t.Text, t.Overlay0,
		t.Text, t.Subtext0, t.Subtext0, t.Overlay0,
		t.Blue, t.Blue,
		t.Green, t.Yellow, t.Red,
		t.Surface1, t.Overlay1, t.Surface0)
}

func hexRGB(hex string) string {
	r, g, b, ok := hexToRGB(hex)
	if !ok {
		return "0, 0, 0"
	}
	return fmt.Sprintf("%d, %d, %d", r, g, b)
}
```

> Note: `hexToRGB` already exists in render.go — do not duplicate it.

## Rollback

If Vesktop path differs from expected:
1. Check actual path: `ls ~/.config/vesktop/`
2. Update `filepath.Join` in `BuildWrites()` accordingly
3. Rebuild + reapply

If CSS vars don't match current Discord version:
- Open DevTools in Vesktop (Ctrl+Shift+I) → inspect `--background-primary` on body
- Update the CSS property names in `renderVesktop()`
