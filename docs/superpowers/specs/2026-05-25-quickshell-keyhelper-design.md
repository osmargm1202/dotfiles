# Quickshell Keyhelper Design

## Goal

Replace the broken Waybar `?` / `Win+/` keybinding helper with a Quickshell panel that renders cached shortcut data generated by `orgm-hypr`.

## Context

Current state:

- Waybar has `custom/keybindings_help` showing `?` and running `orgm-hypr menu keybindings`.
- Hyprland binds `SUPER + slash` to `orgm-hypr menu keybindings`.
- The current helper path is not working reliably for the user.
- Quickshell already exists in the dotfiles for calendar and wallpaper picker, but modules are still organized as separate top-level folders.

User intent:

- Use `orgm-hypr` as the backend for desktop helper data.
- Use Quickshell as the frontend renderer.
- Avoid a permanent helper daemon for now.
- Make the UI legible per host through `HELPER_SCALE=1.00`.
- Start organizing Quickshell modules so future work can grow into a robust project.

## Approved UI Direction

Use the selected **Option B: panel with categories**.

The panel should look like a compact overlay:

- Left column: shortcut categories.
- Right area: shortcut rows for the selected category.
- Header: title, current shortcut (`Win+/`), and close hint.
- Rows: action description, key chord, optional command.
- Keyboard support: `Esc` closes; arrows navigate category/rows if implemented in the first pass; mouse selection is acceptable for initial implementation.

This is preferred over a single square grid because it scales better as shortcuts grow.

## Backend Design

Add a helper command group to `orgm-hypr`:

```bash
orgm-hypr helper init
orgm-hypr helper toggle
```

`orgm-hypr helper init` writes a cached JSON file under:

```text
${XDG_STATE_HOME:-~/.local/state}/orgm-helper/keybindings.json
```

The JSON should be generated from Go code, not by parsing QML or running Quickshell. The initial data source can reuse and improve `internal/menu.KeybindingEntries`, but should expose categories so Quickshell can render a sidebar.

`orgm-hypr helper toggle` should:

1. Ensure the JSON exists by running the same generation path as `helper init` when missing.
2. Write a request file under:

```text
${XDG_STATE_HOME:-~/.local/state}/orgm-helper/keyhelper-request.json
```

3. Launch Quickshell for the keyhelper module if it is not already running, following the existing calendar/wallpaper pattern.

No permanent `orgm-hypr helper daemon` is required for this version. A daemon can be added later if automatic regeneration becomes valuable.

## Data Contract

The cache JSON should be explicit and stable:

```json
{
  "schemaVersion": 1,
  "generatedAt": "2026-05-25T12:00:00Z",
  "defaultCategory": "launchers",
  "categories": [
    {
      "id": "launchers",
      "title": "Launchers",
      "icon": "󰀻",
      "entries": [
        {
          "key": "Win+/",
          "description": "Atajos Hyprland",
          "command": "orgm-hypr helper toggle"
        }
      ]
    }
  ]
}
```

Quickshell should tolerate missing optional fields and show an empty/error state if the JSON cannot be read.

## Quickshell Organization

Introduce a module-oriented layout:

```text
config/shared/.config/quickshell/
  modules/
    keyhelper/
      shell.qml
  components/
  lib/
```

For this task, only `modules/keyhelper/shell.qml` must be created. `components/` and `lib/` can be added as empty directories only if useful to keep the structure visible; no speculative abstractions are required.

Existing modules (`calendar`, `wallpaper-picker`) do not need to be moved in this change. The new structure establishes the path for future modules without risking regressions.

## Host Scale

Add `HELPER_SCALE=1.00` to host-specific environment configuration for `orgm`.

Quickshell should read:

```qml
Quickshell.env("HELPER_SCALE") || "1.00"
```

and apply it to the panel dimensions, spacing, and font sizes through a root `scaleFactor` property. Invalid values should fall back to `1.00`.

## Waybar and Keybinding Integration

Update:

- `config/shared/.config/hypr/lua/keybindings.lua`
- `config/shared/.config/waybar-hypr/config`

Expected behavior:

- `Win+/` runs `orgm-hypr helper toggle`.
- Waybar `?` left click runs `orgm-hypr helper toggle`.
- Waybar `?` right click runs `orgm-hypr helper init`.
- Waybar tooltip explains both actions.

## Error Handling

- If Quickshell is missing, `orgm-hypr helper toggle` should return a clear error.
- If the cache is missing, `helper toggle` should regenerate it before requesting the UI.
- If cache parsing fails in Quickshell, the panel should show an error message instead of rendering blank.
- `helper init` should write JSON atomically to avoid partial reads.

## Testing

Add Go tests around the backend behavior:

- Helper JSON contains categories and entries.
- `helper init` writes valid JSON to an overrideable state directory.
- `helper toggle` creates cache when missing and writes a request file.
- CLI rejects invalid helper subcommands with a usage error.

Manual verification:

- `go test ./...`
- `orgm-hypr helper init`
- `orgm-hypr helper toggle`
- `orgm-dot diff --host orgm`
- After sync/reload, `Win+/` and Waybar `?` open the Quickshell panel.

## Non-Goals

- No always-running helper daemon in this version.
- No automatic parsing of `keybindings.lua`.
- No migration of existing calendar/wallpaper modules to the new `modules/` layout in this change.
- No full visual keyboard map in this version.
