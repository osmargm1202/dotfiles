# Design: orgm-hypr-theme-system

## Architecture

Use `orgm-hypr` as one typed theme engine. Keep shell only as app reload glue where unavoidable.

```text
orgm-hypr theme
  registry loader -> palette resolver -> plan builder -> target renderers -> atomic writer -> reload runner
```

Phase 2 Quickshell talks only to CLI/status JSON.

## Data model

`themes.json` path:

```text
config/shared/.config/orgm-hypr/themes.json
```

Proposed structure:

```json
{
  "schemaVersion": 1,
  "activeDefault": "neutral",
  "themes": [
    {
      "id": "neutral",
      "name": "Neutral",
      "defaultMode": "dark",
      "wallpaper": { "mode": "current", "deriveColors": false },
      "palettes": {
        "dark": { "background": "#...", "foreground": "#...", "accent": "#..." },
        "light": { "background": "#...", "foreground": "#...", "accent": "#..." }
      },
      "targets": { "chromium": { "mode": "export" }, "zen": { "mode": "export" } }
    }
  ]
}
```

Internal palette should normalize to semantic colors: `background`, `surface`, `surfaceAlt`, `foreground`, `muted`, `accent`, `accent2`, `border`, `urgent`, `success`, plus RGB/RGBA helpers.

## Target renderer design

Each renderer exposes:

```text
Name()
Plan(theme, mode, env) -> writes + reloads + warnings
Render(plan) -> byte outputs
```

Initial renderers:

- `hypr`: generated `colors.conf`/Lua-readable palette file.
- `waybar`: generated `theme.css` imported by `style.css`.
- `nwg-dock`: generated CSS or current symlink/include where supported.
- `gtk`: `settings.ini` updates plus CSS variables.
- `qt`: `qt5ct`/`qt6ct` color files and icon/theme fields.
- `launcher`: fuzzel/rofi colors.
- `terminal`: kitty theme include.
- `tui`: yazi/helix theme pointer/generated file.
- `browser`: Chromium extension/theme export, Zen/Firefox export notes.

## Safe write policy

- Build complete apply plan first.
- Dry-run prints all writes, reloads, skipped targets, and warnings.
- Write to temp file in same directory, fsync best-effort, rename atomically.
- Only overwrite generated files with header marker `Generated by orgm-hypr theme` or explicitly configured current files.
- Keep last-applied manifest in `$XDG_STATE_HOME/orgm-hypr/theme/last-apply.json`.
- Store cache artifacts under `$XDG_CACHE_HOME/orgm-hypr/theme`.

## Wallpaper-derived colors

Do not port HyDE Wallbash wholesale. Start with a Go palette interface:

```text
PaletteSource: static | current-wallpaper | image-path
ColorBackend: built-in average/brightness first; optional external matugen/wallust later
```

For first implementation, use current config palette for `neutral` and support `auto` mode by brightness detection. Add richer extraction later if review budget demands.

## Reload policy

Reloads are target-specific and best-effort:

- Hyprland: `hyprctl reload` when session exists.
- Waybar: signal/restart existing known process only when safe.
- nwg-dock: restart helper if configured.
- GTK/Qt: update files and report restart hints; optional `gsettings` when available.
- Browsers: export theme; automatic profile mutation deferred unless explicitly configured.

## HyDE adaptation

Copy concepts:

- central registry;
- generated current files;
- dark/light/auto;
- per-target templates;
- Chromium theme manifest export;
- cache/state separation.

Do not copy:

- global shell environment coupling;
- direct `sed -i` live mutations;
- hyprlang-only assumptions;
- Flatpak override side effects by default.

2. Core plan/renderer/writer tests.
3. Core desktop targets.
4. App targets.
5. Browser export and docs.
